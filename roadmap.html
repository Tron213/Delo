1 Git + Django проект
  • Создать репозиторий StepWise на GitHub (приватный)
  • Установить Django, создать venv, django-admin startproject
  • Первый коммит: .gitignore, requirements.txt
  • python manage.py runserver — видим ракету
    • 💡 Коммитить каждый день. Git — твой удалённый офис.
    2 Приложение core + первая модель User
  • python manage.py startapp core
  • Зарегистрировать в INSTALLED_APPS
  • Django использует встроенную модель User (пока хватит)
  • Настроить подключение к SQLite (по умолчанию)
  • Коммит: "core app added"
    • 📘 Пользователи уже есть — спасибо Django.
    3 Аутентификация: логин, логаут, регистрация
  • Использовать встроенные django.contrib.auth
  • Создать шаблоны registration/login.html (Bootstrap)
  • URL для login, logout, регистрацию написать свою view
  • После входа редирект на /dashboard
  • Коммит: "auth with bootstrap"
    • 🔐 Django делает авторизацию из коробки.
    4 Модель Roadmap (роадмапа)
  • В models.py: class Roadmap (title, description, goal_date, owner = ForeignKey(User))
  • Сделать миграции, применить
  • Зарегистрировать в admin.py
  • Создать суперпользователя, проверить в админке
  • Коммит: "roadmap model"
    • 🗺️ goal_date — необязательно, для дедлайна цели.
    5 Модель Stage (этап) и Task (задача)
  • Stage: roadmap (ForeignKey), title, order
  • Task: stage (ForeignKey), text, completed (bool), order
  • Миграции, регистрация в админке
  • Коммит: "stage and task models"
    • 📌 order — чтобы сортировать этапы и задачи.
    6 Дашборд (список роадмап) + Bootstrap карточки
  • View dashboard: list of user's roadmaps
  • Шаблон с Bootstrap grid, карточки роадмап
  • Кнопка "Создать роадмапу" (форма на отдельной странице)
  • URL /dashboard, защита login_required
  • Коммит: "dashboard UI"
    • 🎨 Адаптив из коробки благодаря Bootstrap.
    7 Создание / редактирование роадмапы (CBV или FBV)
  • CreateView / UpdateView для Roadmap
  • Форма с полями title, description, goal_date
  • После сохранения — редирект на страницу редактирования этапов/задач
  • Коммит: "roadmap CRUD ready"
    • ✏️ Используем Django формы — быстро и безопасно.

    🎯 Полностью рабочий редактор с визуализацией прогресса
    8 Страница одной роадмапы (детали + этапы)
  • View roadmap_detail — отображает заголовок, описание
  • Ниже — список этапов (пока без задач)

    9 Ajax: добавление / удаление этапов (без перезагрузки)
  • Написать view для создания этапа (принимает title, roadmap_id)
  • JS fetch отправляет POST, обновляет список этапов на странице
  • То же для удаления
  • Коммит: "stages via fetch"
    • 🧠 Учимся работать с JSON и обновлять DOM.
    10 Задачи под этапами + чекбоксы (completed)
  • Отображать задачи под каждым этапом
  • Чекбокс, который через PATCH (или отдельную view) меняет completed
  • При клике на чекбокс — отправка fetch, обновление состояния
  • Коммит: "tasks with checkboxes"
    • ✅ После изменения — пересчитываем прогресс.
    11 Прогресс-бары (Bootstrap)
  • Для каждого этапа: процент выполненных задач
  • Общий прогресс всей роадмапы (всего задач / выполненных)
  • Отображать прогресс-баром и текстом
  • Обновлять прогресс при каждом изменении чекбокса
  • Коммит: "progress bars dynamic"
    • 📊 Визуализация — главная фишка.
    12 Добавление / редактирование задач на странице
  • Кнопка "+ задача" под этапом (модальное окно BS)
  • Редактирование текста задачи (иконка карандаша)
  • Удаление задачи
  • Коммит: "task CRUD frontend"
    • ✏️ Учимся работать с модалками и формами.
    13 Сортировка этапов и задач (Drag & Drop)
  • Подключить SortableJS
  • При изменении порядка — отправлять новые order на сервер
  • Создать API-вью для обновления порядка (принимает список id)
  • Коммит: "reorder with sortable"
    • 🔄 Меняй план мышкой — крутая фича.
    14 Завершение недели: полировка, проверка прав
  • Убедиться, что редактировать роадмапу может только владелец
  • Добавить сообщения об успехе (toast)
  • Проверить на мобильном через Bootstrap
  • Коммит: "week2 full editor ready"
