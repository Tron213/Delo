НЕДЕЛЯ 1
🔐 Git, регистрация, UI на Bootstrap
🎯 Репозиторий + работающая авторизация + адаптивный интерфейс
1 Репозиторий и окружение
  • Создать репозиторий StepWise на GitHub (приватный)
  • git clone, создать venv, fastapi, uvicorn, sqlalchemy, python-dotenv
  • .gitignore (venv, .env, __pycache__)
  • Сделать первый коммит: "init project"
    • 💡 Git — твой удалённый офис. Коммиты каждый день, даже мелкие.
    2 Модель User + регистрация
  • Таблица User: id, username, hashed_password
  • Эндпоинт POST /register (хеш пароля bcrypt)
  • Проверить через /docs (FastAPI автоматически)
  • Коммит: "user model and registration"
    • 📘 passlib[bcrypt] — хеширование безопасно.
    3 JWT логин + защита
  • POST /login → возвращает access_token (python-jose)
  • Зависимость get_current_user (Header Authorization)
  • Тестовый эндпоинт /users/me (возвращает username)
  • Коммит: "jwt auth added"
    • 🔒 SECRET_KEY в .env, срок токена 7 дней.
    4 Bootstrap 5 — страница входа/регистрации
  • Подключить Bootstrap CDN, создать шаблон login.html + register.html
  • Адаптивная форма (карточка, центрирование, грид)
  • JS отправка fetch, сохранение токена в localStorage
  • Коммит: "bootstrap auth UI"
    • 🎨 Адаптив из коробки — Bootstrap решает проблему мобильных устройств.
    5 Модель Roadmap + API
  • Roadmap: id, title, description, goal_date, owner_id
  • POST /roadmaps, GET /roadmaps (список моих)
  • GET /roadmaps/{id} (проверка владельца)
  • Коммит: "roadmap crud start"
    • 🗺️ goal_date — необязательно, для дедлайна цели.
    6 Дашборд (список роадмап) на Bootstrap
  • Страница /dashboard — отображение карточек роадмап
  • Кнопка "Создать роадмапу" (модальное окно BS5)
  • Загрузка списка через fetch после логина
  • Коммит: "dashboard with bootstrap cards"
    • 📱 Карточки автоматически адаптируются под экран.
    7 Модель этапа (Stage) + базовая вёрстка
  • Stage: id, roadmap_id, title, order_index
  • API: добавление этапа, получение этапов роадмапы
  • Подключить страницу roadmap_editor.html (заготовка)
  • Коммит: "stages and editor layout"
    • 📌 Порядок этапов важен для визуального таймлайна.
    НЕДЕЛЯ 2
    ✅ Задачи, чекбоксы, прогресс-бары
    🎯 Полноценный редактор с отслеживанием выполнения
    8 Модель Task + API
  • Task: id, stage_id, text, completed (bool), order_index
  • POST /tasks, GET /stages/{id}/tasks, PATCH /tasks/{id} (toggle completed)
  • DELETE /tasks/{id}
  • Коммит: "tasks api"
    • ✍️ completed по умолчанию false.
    9 Страница редактора: рендер этапов и задач
  • Загружаем роадмапу → этапы → задачи (группировка по stage)
  • Bootstrap аккордеон или карточки этапов
  • Для каждого этапа отображаем список задач с чекбоксом
  • Коммит: "render stages & tasks"
    • 🎯 Чекбокс меняет completed через PATCH без перезагрузки.
    10 Прогресс этапа и общий прогресс
  • При загрузке считаем % выполнения по каждому этапу
  • Добавить прогресс-бар Bootstrap (width = %)
  • Общий прогресс всей роадмапы (все задачи / все задачи)
  • Коммит: "progress bars"
    • 📊 Визуализация мотивирует.
    11 Редактирование inline (названия этапов, задач)
  • Кнопка "ред." у этапа → модальное окно (Bootstrap) изменение title
  • У задач: редактирование текста через иконку карандаша
  • Обновление через соответствующие PUT-запросы
  • Коммит: "inline editing"
    • ✏️ UX без перегрузки страницы.
    12 Добавление / удаление этапов и задач
  • Кнопка "+ Этап" — создание нового этапа (ввод названия)
  • Кнопка "+ задача" под этапом
  • Удаление этапа → удаляются все задачи (каскад в БД)
  • Коммит: "add/delete items"
    • 🧹 Подтверждение удаления — через confirm.
    13 Сортировка этапов и задач (Drag & drop простой)
  • Использовать SortableJS (легкая либа) для изменения order_index
  • После изменения порядка → отправить batch update или отдельно PUT stages
  • То же для задач внутри этапа
  • Коммит: "reorder with sortablejs"
    • 🔄 Меняй план мышкой.
    14 Полировка + проверка прав доступа
  • Убедиться, что чужая роадмапа не редактируется (403)
  • Добавить тост-уведомления (Bootstrap Toasts)
  • Пуш в GitHub (регулярные коммиты за неделю)
  • Коммит: "week2 done — full editor"
    • 🚀 Теперь редактор полностью рабочий.

    15 Модель ShareToken + генерация ссылки
  • ShareToken: id, roadmap_id, token (uuid4), created_at
  • POST /roadmaps/{id}/share → возвращает /public/{token}
  • DELETE /share/{token} — отозвать ссылку
  • Коммит: "share tokens"
    • 🔗 Токен уникальный, нельзя подобрать.
    16 Публичная страница (view-only)
  • Эндпоинт GET /public/{token} отдаёт HTML
  • Рендер этапов и задач ТОЛЬКО для чтения (без чекбоксов, без кнопок удаления)
  • Показывать прогресс-бары, но заблокировать изменение
  • Коммит: "public readonly view"
    • 👀 Друзья видят твой прогресс, но не могут менять.
    17 UI: кнопка "Поделиться" + QR
  • В редакторе кнопка "Поделиться" — показывает модалку со ссылкой
  • Генерация QR-кода (qrcode.js) прямо на странице
  • Кнопка копирования ссылки в буфер
  • Коммит: "share modal + qr"
    • 📱 Друг сканирует телефоном → видит план.
    18 Подготовка к деплою: .env, requirements, wsgi
  • Обновить .env.example, проверить SECRET_KEY, DATABASE_URL
  • Заморозить requirements.txt (pip freeze)
  • Настроить запуск через uvicorn production (workers)
  • Коммит: "ready for deploy"
    • ⚙️ DATABASE_URL переключить на PostgreSQL (опционально).
    19 Деплой на VPS: systemd
  • Загрузить код на сервер (git pull или rsync)
  • Установить зависимости, создать .env
  • Написать юнит /etc/systemd/system/stepwise.service
  • Запустить, проверить journalctl
  • Коммит: "deploy systemd"
    • 🐧 Сервис живёт даже после выхода из SSH.
    20 nginx reverse proxy + статика
  • Установить nginx, настроить server блок proxy_pass на 127.0.0.1:8000
  • Отдать статические файлы через nginx (если есть)
  • Проверить доступ без порта 8000
  • Коммит: "nginx config"
    • 🌍 Твой сайт доступен на 80 порту по IP.
    21 SSL (certbot) + автообновление
  • Установить certbot, получить сертификат для домена
  • Настроить автоматическое обновление (systemd timer)
  • Проверить публичную ссылку через HTTPS
  • Коммит: "ssl https final"
    • 🔒 Безопасно и современно.
